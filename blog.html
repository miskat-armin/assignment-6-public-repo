Discuss the scope of var, let, and const
var: Function-Scoped: Variables declared with var are function-scoped, which means they are only accessible within the function in which they are declared. If declared outside of any function, they become globally scoped, accessible throughout the entire program. Hoisting: Variables declared with var are hoisted to the top of their containing function or global context, which means the variable declaration is moved to the top of its enclosing scope during the compilation phase. However, the assignment remains in place. This can lead to unexpected behavior. Reassignment:You can reassign and redeclare a variable declared with var within the same scope. let: Block-Scoped: Variables declared with let are block-scoped, meaning they are limited in scope to the block or statement in which they are defined. They are not hoisted to the top of the function or global context. No Re-declaration: You cannot redeclare a variable within the same scope using let, but you can reassign its value. const: Block-Scoped: Like let, variables declared with const are block-scoped. Constant Value: Variables declared with const are constants, meaning their value cannot be reassigned after declaration. However, if the variable holds an object or array, the properties or elements of that object or array can still be modified.
Tell us the use cases of null and undefined
null: Explicit Absence of a Value: null is often used to explicitly represent the absence of a value or the intentional empty state of an object or variable. It is typically assigned by developers to indicate that a variable or object property should have no value or reference to an object.

let someValue = null; // Explicitly indicating that 'someValue' has no meaningful value.

Clearing Object References: Developers may set an object property to null to clear its reference, which can help with memory management in situations where the object is no longer needed.

let myObject = { data: 'some data' };
myObject = null;

undefined: Implicit Absence of a Value: undefined represents the absence of an assigned value. In most cases, it occurs when you declare a variable but don't initialize it or when you access a non-existent object property or function argument.

let someVar;
let obj = {};
console.log(obj.nonExistentProperty);

Function Return Values: When a function doesn't explicitly return a value, it implicitly returns undefined.

function doSomething() {
// No return statement, so 'undefined' is returned.
}

Function Parameters: If a function is called with fewer arguments than declared in its parameters, the missing parameters are assigned the value undefined.

function greet(name) {
console.log(`Hello, ${name}!`);
}
greet();

What do you mean by REST API?

A REST API (Representational State Transfer Application Programming Interface) is a standardized way for software applications to communicate over the internet using a set of rules and conventions. It allows systems to request and exchange data or perform actions on remote servers. REST APIs are widely used in web development for building scalable and stateless web services, often using HTTP methods like GET, POST, PUT, and DELETE to interact with resources represented as URLs.
